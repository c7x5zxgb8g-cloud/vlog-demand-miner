# NextTake 完整使用手册

> 适用于已经把 NextTake 接入 Codex 或其他支持本地 Agent Skill 的 AI 工具之后。

NextTake 是个人短视频创作者的内容创作与发布优化系统。它负责从同赛道公开内容和评论中发现问题、生成本期文案、记录发布前判断、复盘人工发布后的数据，并生成下一条推荐与下一期完整文案。

平台发布始终由创作者自己完成。NextTake 不会自动上传视频，也不会把需求分或发布前判断当作流量承诺。

## 1. 接入 AI 工具前的要求

完整运行 NextTake 的 AI 工具需要具备以下能力：

1. 能加载包含 `SKILL.md`、`scripts/`、`references/` 和 `vendor/` 的完整 Skill 目录；
2. 能在本机执行 Python 命令；
3. 能读取和写入你指定的研究项目与创作者项目目录；
4. 能把生成的 Creator Studio HTML 绝对路径返回给你；
5. 在需要平台登录时，允许你自己完成二维码、浏览器登录或验证码操作。

只支持上传提示词、但不能执行本地脚本或读写文件的聊天工具，不能运行完整 NextTake。它最多只能参考方法生成文案，无法保存证据、预测、复盘和下一期内容。

当前仓库已验证的接入方式是 Codex Skill。其他支持 Agent Skill 的 AI 工具，应按照该工具的 Skill 安装说明导入整个 `vlog-demand-miner/` 目录，不能只导入 `SKILL.md`。

### Codex 安装

在仓库根目录执行：

```bash
mkdir -p "$HOME/.codex/skills/vlog-demand-miner"
rsync -a \
  --delete \
  --exclude '__pycache__' \
  --exclude '*.pyc' \
  "$PWD/vlog-demand-miner/" \
  "$HOME/.codex/skills/vlog-demand-miner/"
```

同步后重新打开一个 Codex task，并在对话中使用 `$vlog-demand-miner`。

## 2. 第一次接入后的验收

不要一开始就登录真实平台。先让 AI 运行离线演示，确认 Skill、Python 脚本和 HTML 输出都能工作。

对 AI 说：

```text
请使用 $vlog-demand-miner 运行 NextTake 离线演示。
不要连接真实平台，也不要安装采集环境。
完成后告诉我：内容机会数量、本期文案标题、下一期文案标题，
并返回 Creator Studio 的 HTML 绝对路径。
```

AI 应该返回：

- `status: ok`；
- 4 个团播内容机会；
- 本期文案《做团播到底能不能赚钱？先把这四笔账问清楚》；
- 下一期文案《面试团播公司，先反问这 7 个问题》；
- 一个可以在浏览器中打开的 `index.html` 绝对路径。

Studio 中应当能看到“发现、创作、复盘、下一条”四个页面区域，并同时包含本期文案和下一期文案。演示中的发布及表现数据必须标记为“演示数据”。

如果这一步失败，先处理“常见问题”中的安装或路径问题，不要进入真实采集。

## 3. 理解两个项目目录

真实使用时，建议把市场研究数据和个人创作数据分开保存：

```text
~/nexttake/
├── research/<赛道名称>/     # 公开视频、评论、Evidence 和内容机会
└── creator/<账号名称>/      # 文案、预测、发布记录、复盘和受众画像
```

例如：

```bash
mkdir -p "$HOME/nexttake/research/group-live"
mkdir -p "$HOME/nexttake/creator/xiaolin"
```

后面的手册使用：

```text
研究项目：~/nexttake/research/group-live
创作者项目：~/nexttake/creator/xiaolin
```

实际对 AI 下达任务时，尽量使用绝对路径。这样即使 AI 工具的当前工作目录发生变化，也不会把研究数据写错位置。

## 4. 初始化创作者项目

这是每个账号第一次使用时必须完成的步骤。一个创作者项目对应一个账号或一套明确的内容定位。

对 AI 说：

```text
请使用 $vlog-demand-miner 初始化创作者项目。

项目目录：/Users/<你的用户名>/nexttake/creator/xiaolin
内容方向：团播求职避坑
主要平台：抖音
内容形式：单人口播
典型时长：60 到 90 秒
发布节奏：每周 3 条
目标受众：准备进入团播行业的年轻求职者
历史样本：没有，请按冷启动处理

完成后告诉我创建了哪些文件，以及下一步需要什么信息。
```

如果你没有一次性提供完整信息，NextTake 会继续询问内容类型、典型时长、发布节奏和历史样本。

初始化完成后，创作者目录通常包含：

```text
audience.md       # 当前受众画像
candidates.md     # 待创作候选池
rubric_notes.md   # 评分规则与长期观察
scripts/          # 本期和后续完整文案
predictions/      # 发布前判断与发布后校验
videos/           # 已拍摄、已发布和复盘记录
samples/          # 用户提供的历史样本
```

看到 `creator_init_required` 说明项目尚未初始化，重新执行本步骤即可。

## 5. 选择内容机会的来源

NextTake 支持两种开始方式。

### 方式 A：已经有内容机会

适合团队已经做过研究，或研究项目中已经存在 `OPP-...` 内容机会的情况。

对 AI 说：

```text
请使用 $vlog-demand-miner 检查研究项目：
/Users/<你的用户名>/nexttake/research/group-live

列出最新的内容机会，按需求信号强弱展示：
1. 受众问题；
2. 观众原话或证据；
3. 支持证据数量；
4. 反证和样本限制；
5. 对应的 OPP ID。

先不要生成文案，等我选择。
```

选择后说：

```text
选择内容机会 OPP-<ID>。
把它导入创作者项目：
/Users/<你的用户名>/nexttake/creator/xiaolin

完成后返回 candidate ID、source pack 路径和下一步建议。
```

### 方式 B：从真实同赛道账号开始

适合还没有研究结果，需要从公开视频和公开评论寻找内容机会的情况。

第一次真实采集前，对 AI 说：

```text
请使用 $vlog-demand-miner 检查并配置本机采集环境。

研究项目：/Users/<你的用户名>/nexttake/research/group-live
需要的平台：抖音

先执行环境检查。只有缺少必要依赖时才安装。
需要登录、扫码、验证码或浏览器确认时立即暂停，让我手动完成。
不要读取、复制或输出 Cookie、Token 和浏览器存储。
```

环境准备完成后，可以选择全自动检索或自己提供对标账号。

#### 全自动检索模式

对 AI 说：

```text
请使用 $vlog-demand-miner 根据赛道自动寻找对标账号。

研究目录：/Users/<你的用户名>/nexttake/research/first-renter
赛道：首次租房避坑
平台：抖音、B站
每个平台最多登记 3 个候选账号。

每个平台只搜索第 1 页，所有搜索严格串行并保留随机缓冲。
先返回选中的账号、匹配关键词和选择依据，再继续单页同步。
搜索相关性不等于赛道需求，不要输出流量保证。
```

B站可以直接使用本地 CLI 搜索。抖音自动搜索需要 Browser Provider 和一次人工登录；抖音 Sidecar继续负责已知账号的作品与评论采集。需要登录、验证码或风险验证时，AI必须暂停。

#### 用户提供账号模式

提供 2 到 5 个明确的主页链接、分享链接或账号 ID。对 AI 说：

```text
请使用 $vlog-demand-miner 初始化同赛道研究项目。

研究目录：/Users/<你的用户名>/nexttake/research/group-live
研究名称：团播求职内容机会
目标受众：准备进入团播行业的年轻求职者

导入以下对标账号：
- 平台：抖音；名称：账号 A；主页或分享链接：<抖音 URL>
- 平台：B站；名称：账号 B；主页链接：https://space.bilibili.com/<UID>

先做小样本研究：每个账号最多同步 1 页，并选择 6 条相关作品。
采集公开作品信息和公开评论；需要字幕时先告诉我。
遇到登录失效、验证码、风控或接口结构变化时停止，不要反复重试。
```

采集完成后继续说：

```text
请继续分析同赛道内容和评论：
1. 将视频文本与评论分开提取 Evidence；
2. 聚类为具体的受众问题；
3. 对内容机会排序；
4. 每个机会展示观众原话、支持证据、反证和样本限制；
5. 生成研究报告，但暂时不要写文案。
```

研究结果是选题假设，不代表整个赛道，也不保证播放量。账号和作品数量较少时，应先把结论用于试拍，而不是当作市场定论。

## 6. 生成第一期完整文案

确定内容机会并导入创作者项目后，要求 NextTake读取 source pack 生成文案。

对 AI 说：

```text
请使用 $vlog-demand-miner，根据刚才导入的内容机会生成本期完整文案。

创作者项目：/Users/<你的用户名>/nexttake/creator/xiaolin
目标时长：60 到 90 秒
表达方式：直接、具体，不夸大收入，不替观众做职业决定

文案需要包含：
1. 开头 Hook；
2. 完整口播正文；
3. 可以拍摄的段落结构；
4. 结尾互动问题；
5. 使用了哪些 Evidence；
6. 当前证据的限制。

完成后返回文案标题和 scripts/ 下的文件路径。
```

如果文案需要修改，直接针对文件继续说：

```text
修改 scripts/<draft>.md：
- 开头 3 秒更直接；
- 删除无法由现有 Evidence 支持的结论；
- 正文控制在 75 秒左右；
- 保留结尾的评论引导。

修改后列出发生变化的段落。
```

在你确认最终稿之前，不要启动发布前预测。

## 7. 打分和发布前判断

最终稿确认后，对 AI 说：

```text
请使用 $vlog-demand-miner：
1. 给本期文案 scripts/<draft>.md 打分；
2. 说明最弱的两个维度和可以执行的修改建议；
3. 如果没有阻断问题，启动发布前预测；
4. 保存预测文件并返回路径。

不要把评分或预测描述成流量承诺。
```

发布前判断的作用是留下可验证的假设。例如：

- 哪个 Hook 可能提高停留；
- 哪个清单可能带来收藏；
- 哪个问题可能引发评论；
- 哪些判断因为样本少而置信度有限。

预测必须在看到发布结果之前保存。发布后只能追加复盘，不能把原预测改成“看起来正确”。

## 8. 登记拍摄和人工发布

拍摄完成后，对 AI 说：

```text
请使用 $vlog-demand-miner 登记已拍摄：
scripts/<draft>.md

最终拍摄稿与当前文案一致。
```

如果现场改过稿，应告诉 AI 实际拍摄时改了什么，确保最终拍摄稿与真正发布的内容一致。

然后由你自己在抖音、B站、小红书或其他平台发布。NextTake 不会代替你上传视频。

发布后说：

```text
请使用 $vlog-demand-miner 登记已发布。

视频对应：scripts/<draft>.md
平台：抖音
公开视频链接：<URL>
发布时间：2026-07-20 20:00 +08:00

只登记发布结果，不要调用平台上传接口。
```

## 9. 导入播放、互动和评论数据

建议在固定观察窗口导入数据，例如发布后 72 小时。不同视频尽量使用相同观察窗口，才便于长期比较。

最简单的方式是直接把原始数据交给 AI：

```text
请使用 $vlog-demand-miner 导入以下原始表现数据，并复盘本期内容。

对应视频：videos/<video-folder>/
观察时间：发布后 72 小时

{
  "views": 12840,
  "likes": 1126,
  "comments": 143,
  "shares": 216,
  "saves": 394,
  "follows": 87,
  "completion_rate": 0.38,
  "captured_at": "2026-07-23T20:00:00+08:00",
  "top_comments": [
    "能不能讲讲底薪、提成和流水到底怎么算？",
    "下一期想看怎么判断公司靠不靠谱。"
  ]
}

要求：
1. 保存原始指标；
2. 自动计算互动比例；
3. 对照发布前预测，标记为验证、推翻或暂时无法判断；
4. 提取评论中的下一步需求；
5. 不修改原始发布前预测。
```

只提交内容分析需要的数据。不要把以下信息发给 AI 或写入项目：

- 评论者平台用户 ID；
- 昵称、头像、电话或其他联系方式；
- Cookie、Token、请求头或浏览器存储；
- 平台私有响应；
- 自己计算的 `likes_per_view` 等比例字段。

如果当前平台 Adapter 没有完成真实账号验证，优先手工粘贴原始指标和匿名评论，不要把“源码存在”理解为已经可以自动抓取。

## 10. 更新受众并生成下一期文案

复盘完成后，对 AI 说：

```text
请使用 $vlog-demand-miner 完成这一轮的后续动作：
1. 根据本期表现和匿名评论更新受众画像；
2. 推荐下一条内容；
3. 说明推荐引用了哪些本期评论或原始 Evidence；
4. 说明下一条为什么与本期不同；
5. 直接生成下一期完整文案；
6. 返回推荐文件、受众画像和下一期文案的路径。
```

一个完整周期必须得到两份不同阶段的文案：

```text
第一期完整文案
  -> 人工拍摄和发布
  -> 导入真实表现
  -> 复盘与受众更新
  -> 下一条推荐
  -> 下一期完整文案
```

如果 AI 只返回一个选题标题，没有生成第二份完整稿，继续说：

```text
请根据刚才的下一条推荐继续执行“生成下一期文案”，
写入 scripts/，并返回完整文案和文件路径。
```

## 11. 生成 Creator Studio

完成本期复盘和下一期文案后，对 AI 说：

```text
请使用 $vlog-demand-miner 为当前创作周期生成 Creator Studio。

研究项目：/Users/<你的用户名>/nexttake/research/group-live
创作者项目：/Users/<你的用户名>/nexttake/creator/xiaolin

请自动登记本期文案、发布前判断、原始表现、复盘、受众画像、
下一条推荐和下一期文案，然后生成只读 Studio。

完成后返回 index.html 的绝对路径，并确认页面同时包含：
发现、本期文案、本期复盘、下一条推荐、下一期完整文案。
```

Creator Studio 是静态只读页面。它适合个人复盘、团队同步和路演展示，但不会修改项目文件，也不会自动发布内容。

## 12. 日常常用说法

### 查看当前进度

```text
请使用 $vlog-demand-miner 查看创作状态。
告诉我当前正在做哪一期、缺少哪个步骤，以及下一步应该说什么。
```

### 继续一个中断的任务

```text
请使用 $vlog-demand-miner 检查当前项目已有产物。
不要重复已经成功的采集或分析，从最近的安全检查点继续。
```

### 导入自己的历史内容

```text
请使用 $vlog-demand-miner 导入我的历史内容样本。
先分析我常用的结构、时长和表达方式，不要直接改写历史稿。
```

### 导入对标账号

```text
请使用 $vlog-demand-miner 导入对标账号。
只学习可观察的内容结构和受众问题，不复制原文，不冒充原作者风格。
```

### 根据赛道自动寻找对标账号

```text
请使用 $vlog-demand-miner 根据“首次租房避坑”自动寻找对标账号。
平台为抖音和B站，每个平台最多 3 个；只搜索第 1 页并串行执行。
选中后返回账号、匹配原因和限制，再开始单页同步。
```

### 只生成选题，不写文案

```text
请使用 $vlog-demand-miner 找出 5 个内容机会。
每个机会必须附 Evidence 和限制，先不要生成文案。
```

### 调整长期评分规则

```text
请使用 $vlog-demand-miner 检查最近多期复盘。
只有存在重复出现的同向证据时，才建议更新评分规则；
单条视频的偶然表现只记录为观察。
```

## 13. 如何判断一轮是否完成

一次完整的 NextTake 创作周期应当同时具备：

- 至少一个有来源和限制说明的内容机会；
- 一份已确认的本期完整文案；
- 一份发布前判断；
- 一次人工拍摄和人工发布登记；
- 一组原始表现数据和匿名评论；
- 一份对照预测的复盘；
- 一份更新后的受众画像；
- 一个引用 Evidence 或评论的下一条推荐；
- 一份下一期完整文案；
- 一个同时展示上述内容的 Creator Studio。

缺少下一期完整文案时，这一轮还没有形成“让上一条决定下一条”的闭环。

## 14. 常见问题

### AI 没有识别 `$vlog-demand-miner`

确认导入的是完整 Skill 目录，不是单独的 `SKILL.md`；然后重新打开 AI task。Codex 用户可重新执行安装章节中的 `rsync`。

### AI 只能回答文字，不能生成文件

当前 AI 工具没有本地命令和文件权限。换用支持 Agent Skill、本地 Python 和工作区读写的工具，或为当前工具开启这些能力。

### 出现 `creator_init_required`

创作者项目还没有初始化，或 AI 指向了错误目录。使用绝对路径重新执行“初始化创作者项目”。

### 出现 `cluster_score_required`

研究项目尚未完成 Evidence 聚类。让 AI 继续完成分析和内容机会聚类，再导入创作者项目。

### 出现 `cluster_not_found`

使用的 `OPP-...` 不属于最新研究结果。重新列出最新内容机会，并选择当前的 OPP ID。

### 出现 `incomplete_nexttake_link`

Creator Studio 所需的本期文案、预测、表现、复盘、推荐或下一期文案还没有登记完整。让 AI 列出缺少的文件，不要伪造缺失数据。

### Creator Studio 没有下一期完整文案

先执行“推荐下一条”，再执行“生成下一期文案”，最后重新生成 Studio。

### 平台要求登录或验证码

让 AI 暂停，由你自己完成登录。NextTake 不应读取、复制、输出 Cookie，也不应自动绕过验证码或风控。

### 采集环境失败，但只想演示

离线演示不依赖真实平台。直接要求 AI“运行 NextTake 离线演示，不连接真实平台”。

### 数据很好，是否说明预测准确

不一定。复盘应逐项比较发布前判断和结果。播放量、收藏、评论和转粉可能支持不同结论，单条视频也不足以更新长期规则。

## 15. 使用边界

- NextTake 生成的是可修改文案，不替代创作者核实事实和承担表达责任；
- NextTake 不自动上传、定时发布或确认平台发布成功；
- 内容机会排序是研究信号，不是流量、收入或商业成功承诺；
- 竞品和评论研究只使用允许访问的公开信息；
- 遇到登录失效、验证码、风控或协议变化时停止，不绕过平台验证；
- 当前重点是个人创作者的内容产出和优化分析，不包含云端多用户协作。
